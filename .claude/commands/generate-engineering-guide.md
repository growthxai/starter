## Rules

- Keep the same level of detail and all code examples from the originals
- Do not add new conventions or opinions — only transform what exists
- Do not summarize or abbreviate — engineers need the same depth as the source files
- Preserve markdown formatting, tables, and code blocks exactly
- When CLAUDE.md and a pattern file cover the same topic, keep the more detailed version and remove the duplicate
- The result should read naturally top-to-bottom as a standalone guide — no dangling references to other files
- **CRITICAL: Do not truncate.** If you notice you're running long, split into another Edit pass rather than cutting content.
